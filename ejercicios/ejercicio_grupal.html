Ejercicio Grupal

Introducción

El 50% de la nota final de la asignatura de Análisis y Visualización de BigData se calcula mediante un ejercicio grupal. Este ejercicio tiene como objetivo poner en práctica las distintas técnicas de recolección de datos presentadas durante las sesiones teórico-prácticas de la asignatura.

El trabajo busca probar la capacidad de los alumnos para desarrollar de forma autónoma un análisis de datos completo, algo que abarca desde la recolección de la información, su limpieza y procesado, hasta la presentación de los datos mediante texto y gráficos interactivos. Se valora la autonomía, la creatividad y originalidad, así como la capacidad para resolver problemas técnicos y metdológicos.

¿En qué consiste el trabajo?

Este ejercicio consiste en crear artículo que presente, mediante texto y gráficos interactivos incrustados, una investigación, caso de estudio o análisis de un fenómeno basado en datos. Se trata de un ejercicio libre, por lo que podéis elegir la temática que os parezca más adecuada. Los grupos son de 2 personas.

Se recomienda elegir un tema en el que sea posible obtener una gran cantidad de datos de un modo relativamente fácil y rápido. Por ejemplo, algo relacionado con una plataforma que tenga una API abierta (Twitch, Spotify, YouTube, etc). Sin embargo, es posible analizar temas con datos procedentes de scrapping o bases de datos alternativas (INE, IDESCAT, etc), o cualquier otra fuente que consideréis oportunda. Buscar los datos y justificar su "cantidad" es parte del trabajo. Los datos que dan lugar al análisis deben ser "masivos". No existe un máximo de datos ni un minimo, pero se valorará que los alumnos manejen gran cantidad de información y sean capaces de vincular los datos con otras fuentes. También se valorará la capacidad de los alumnos para escalar y aumentar la cantidad de datos recopilados.

Para recolectar y tratar los datos, el equipo deberá desarrollar scripts de Python. Este código forma parte del trabajo que se deberá entregar. Se valorará el uso de Pandas, Numpy, así como otras liberías específicas de Python creadas para extraer información de APIS, o bien tratar datos masivos. Aunque sea posible, y en ocasiones incluso más eficiente, se prohibe usar excel como método de manipulación de datos: queremos aprender a usar python. Forma parte del trabajo investigar librerías, encontrar soluciones, valorar y juzgar la idoneidad de las técnicas usadas.

Apartados del trabajo

El trabajo se dividirá en dos partes: la primera será técnica, y la segunda tendrá un carácter divulgativo.

Apartado Técnico:

El apartado técnico del trabajo será una sección de texto donde hablaremos de todo lo relacionado con la elaboración del proyecto. Aquí se exponen las técnicas usadas, el código, las dificultades, limitaciones y posibilidades del trabajo. Debe incluir cosas como:

  1. Presentación del caso de estudio, incluyendo la motivación o interés que existe sobre la cuestión.
  2. Se aconseja exponer uno o varios objetivos concretos: ¿Qué queréis analizar y por qué?
  3. Una explicación detallada del proceso de obtención de datos, incluyendo un link a repositorio de Github con código completo, y fragmentos de código incrustados en el artículo, donde se explique de forma clara cómo se han recolectado los datos.
  4. Una explicación de cómo se han procesado/combinado los conjuntos de datos.
  5. Una presentación del dataset resultante: ¿Qué cantidad de filas tiene? ¿Qué columnas? ¿Qué datos faltan? ¿Qué datos tenemos?.

Apartado divulgativo:

El apartado "divulgativo" del trabajo es una sección donde presentamos la investigación realizada, analizamos los datos y creamos la narración. El texto se puede presentar de muchas maneras, pero no se debe perder de vista que se trata de un texto de extilo "periodístico": queremos explicar una historia, fenómeno o caso a través de datos.

  1. Un texto con estilo periodístico que explique el fenómeno y presente gráficos interactivos para sustentar la información.
  2. Apartado de conclusiones o posibles futuras investigaciones.

¿Cómo se presenta?

El trabajo se presenta en formato .html. Es decir: tenéis que entregar el trabajo como si se tratara de una web muy sencilla en formato "one page". Esta web debe contener los dos apartados anteriormente mencionados. Tenéis un ejemplo aquí: Articulo de ejemplo.

Podéis copiar esta plantilla de ejemplo. El diseño de la web tiene que ser limpio, proporcionado y mínimamente adaptativo para ordenador (no hace falta teléfono). Plantilla html para el articulo (GitHub).

Presentación oral: 11/06/2024

En la presentación oral, los alumnos deberán presentar el proyecto al completo: objetivos de la investigación, cómo se han recolectado los datos, cómo se han procesado, la web creada y los resultados. Luego, los profesores y compañeros podrán hacer preguntas. La presentación es obligatoria, y deberán participar todos los miembros del equipo, salvo casos justificados.

Criterios de Evaluación

    Parte Código: (50%)
  1. Calidad general del código (claro, sencillo, funcional, ordenado).
  2. Soluciones técnicas a los problemas que presentan los datos
  3. Dificultad para la recolección de datos
  4. Código web, calidad de la presentación visual, maquetación, responsive, y url relativas

    5. Parte texto: (50%)
  5. Calidad general del texto.
  6. Calidad de los gráficos y visualizaciones realizados con Tableau.
  7. Capacidad para generar nuevas preguntas y plantear nuevos problemas.
  8. Originalidad, creatividad, capacidad narrativa, etc.
